类和对象

面向对象基本概念

对象是客观世界中的某个具体事物，对象的概念是面向对象技术的核心。

面向对象技术中的对象就是现实世界中某个具体的物理实体在计算机逻辑中的映射和体现，它可以是有形的，也可以是无形的。

比如：电视是一个具体存在的，拥有外形、尺寸、颜色等外部特性(或称属性)和开、关、设置等功能的实体。

从程序设计的角度来看，事务的属性或特性可以用变量来表示，行为或功能则用方法来反映面向对象的程序设计方法就是将客观事物抽象成为“类”，并通过类的“继承”实现软件的可扩充性和可重用性。

类是同种对象的集合与抽象。

在面向对象的程序设计中，定义类的概念来表述同种对象的公共属性和特点。

类是一种抽象的数据类型，它是具有一定共性的对象的抽象，而属于类的某一对象则被称为是类的一个实例，是类的一次实例化的结果。

成员变量与成员方法

成员变量

定义格式如下：

  • 修饰符：表示变量的访问权限（默认访问、public、protected 和 private）
  • static：表示是一个类成员变量（静态变量）；
  • final：表示是一个常量（最终成员变量）；
  • transient：表示一个临时变量；
  • 作用域：整个类。
成员方法

定义格式如下：

  • 修饰符：方法的访问权限（默认的、public、protected 和 private）和一些其他特性（ static、final、abstract 等）；
  • static：类方法（静态方法）；abstract：抽象方法（无方法体的方法）；final：最终方法（不能被子类改变）；
  • 返回类型：执行该方法后返回的数据类型，如果该方法没有返回值，则返回类型必须写 void；
  • throws：表示抛出异常

main 方法

不难看出 main 方法是一种特殊的成员方法。

以下是简单的解释：

  • main 方法是所有 java 应用程序执行的入口，可以运行的 Java 应用程序必须含有 main 方法。

  • main 方法是 JVM（ java 虚拟机）自动调用，需对 JVM 可见，所以 main 方法需要 public 修饰。

    • 同理，main 方法所在的类也需要 public 修饰符。

  • 由于 main 方法是所有程序的入口，也就是 main 被调用时没有任何对象创建，不通过对象调用某一方法，只有将该方法定义为静态方法，所以 main 方法是一个静态方法，既需要 static 修饰。

  • JVM 对于 java 程序已经是最底层，由它调用的方法的返回值已经没有任何地方可去，因此，main 方法返回值为空，既需用 void 修饰。

  • String args[ ] 我们用于在接受命令行传入的参数。

如果没有自定义构造方法，则 Java 调用类的默认构造方法。Java 将使用默认值来初始化成员变量。

如果在类中没有自定义的构造方法，则 Java 调用类的默认构造方法，将使用默认值来初始化成员变量。

对象的清除

当一个对象不再被需要时，便可将空引用（ null ）赋给相应的类变量。

在 Java 中，程序员不需要自己管理所生成对象的内存空间，运行过程中，JVM 通过垃圾收集器周期性地清除无用（即不再被引用的）对象并释放它们所占的内存空间。

子类和父类的静态方法

父类的静态方法不能被子类覆盖为非静态的方法，反之亦然；

子类可以定义与父类的静态方法同名的静态方法，以便在子类中隐藏父类的静态方法

final 关键字

final 修饰类，称为最终类，该类不能被继承

final 修饰方法，该方法不能被其所在类的子类覆盖（重写）

final 修饰方法的参数，表示该方法不期望被传进来的参数有任何改变

final 修饰变量时，完成第一次赋值之后就不能改变了。对 final 变量的初始化可以在变量声明或构造方法中进行。如果在声明中初始化了，则构造方法中不能再次对其赋值。

修饰变量

final 修饰基本变量，即为常量，值无法改变；

final 修饰引用类型变量，该引用变量不能改变，即不能重新赋值，但对象内的成员可以改变

final 修饰的成员变量只能在定义时或在构造方法里赋值，不能在其他地方赋值，且只能赋值一次

import 和包

包

包是 Java 提供的类的组织方式，它是 Java 有效地管理类的一个机制。包的目的是解决类命名重名的问题。

一个包对应一个文件夹，其下面可以包含很多类。

不同包中的类可以取相同的名字。一般在创建项目时为了便于管理，源文件和字节码文件是分开保存在不同的目录中的。

实例：

  1. 项目的文件夹为 myproject；

  2. 源文件保存在 myproject 下的 src 文件夹中；

  3. 字节码文件保存在 myproject 下的 classes 文件夹中

包的定义

格式为：

  • package 语句必须是 Java 源文件的第一条语句；

  • 功能：将源文件中的所有类或接口归类到指定的包中；

  • 包名须符合 Java 标识符的命名规则，当包名由多单词组成时，所有字母通常都采用小写的形式；

  • 包名可理解为一个目录名，它可由多个不同等级依次组成，不同等级的包名之间由 “. ” 相分割。

    • 如：package chap3.syudy.help

  • 如果源程序中省略了 package 语句，编译后将它们的字节码存放在同一个目录项，那么这些类都将属于同一个包，这个包没有包名，因此被叫做无名包。
